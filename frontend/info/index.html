BotBuilder

A desktop app for building Telegram and VK bots without manual project setup.

Main menu sections

Each section is a separate module. Briefly, what it does:

Chat bots

Bot list, tokens, start/stop, open the scenario builder.

Scenario editor

Canvas with blocks and connections.

Plugins

Block catalog: built-in and custom.

Component wizard

Create and edit custom blocks.

Template library

Ready scenarios, import/export .bbpack.json.

Analytics

Activity charts and calendar.

Database

View and edit user_data.db.

Broadcasts

HTML messages with media.

Monitoring

CPU/RAM, bot processes, logs.

Deploy

Docker archive or VPS via SSH.

Quick start

  1. Create a bot in the messenger.
  2. Open Chat bots and add a bot.
  3. Open the scenario builder.
  4. Save the scenario and start the bot.
  5. Test in Telegram or VK.
Tip. For VK you can apply a default scenario from the bot card.

Chat bots

The Chat bots section is the hub for your projects.

What you can do

  • Add and remove bots (Telegram / VK).
  • Start and stop the bot process.
  • Open the visual scenario builder.
  • For VK — verify the token and apply a template scenario.
Important. After changing the scenario you usually need to restart the bot.

Scenario editor

Opens from the bot card. A scenario is a graph of nodes and connections.

How to build logic

  1. Drag a block from the palette onto the canvas.
  2. Click a block to open properties on the right.
  3. Connect one block's output to another's input.
  4. Save the scenario. Version history lets you roll back.

Common blocks

  • Start — entry point.
  • Message — text to the user.
  • Menu — buttons and transitions.
  • Condition — branch by data.
  • Command / data — commands and fields.
  • Custom plugins — your published blocks.

Plugins & blocks

A plugin is a reusable scenario block with UI fields and a Python handler.

View and manage

Open Plugins: list of all blocks.

Creating a custom block

  1. Component wizard → set ID, name, color.
  2. Add property fields — they appear in the editor panel.
  3. Write or edit the Python handler template (Jinja2).
  4. Publish the component — it appears in the palette.

Import and sharing

  • Scenarios ship in .bbpack.json via the Template library.
  • If plugins are missing, the app will warn you.
  • Custom plugins live in the app user data folder.
If plugin save fails (405). Fully restart BotBuilder.

Template library

In Templates — local and marketplace packages.

  • Preview — view the graph before installing.
  • Install on bot — apply to a project.
  • Import / export — share files.
  • Edit template — scenario editor in template mode.

Analytics & database

Analytics

Analytics shows activity for the selected bot.

Database

Database opens the project user_data.db.

Broadcasts

Send HTML messages with media to subscribers.

The bot must be available (valid token).

Monitoring

Monitoring shows load, processes and streaming logs.

Deploy

Deploy prepares the bot to run outside your PC:

  • Docker archive — download a package.
  • VPS via SSH — upload and start the service.

Before deploy, ensure the scenario is saved and the token is current.

Useful tips

  • Help → Application data folder opens the projects directory.
  • Theme: View → Light / Dark or Ctrl+Shift+L.
  • Multiple windows: View → New window.
  • If a page is blank — restart the app and check Settings.
  • After updating BotBuilder, restart bots for the new runtime.
